# 各家云服安全组配置

裸 IP + ws:// 方案只需要开放两个端口：

| 端口 | 协议 | 来源 | 用途 |
|---|---|---|---|
| 22 | TCP | 你的家用 IP 或 0.0.0.0/0 | SSH 运维 |
| 3000 | TCP | 0.0.0.0/0 | 游戏 WebSocket 连接 |

## 阿里云 ECS

1. 控制台 → 云服务器 ECS → 实例 → 你的实例 → 「安全组」标签
2. 点 「配置规则」 → 「入方向」 → 「手动添加」
3. 协议=TCP、端口范围=3000/3000、授权对象=0.0.0.0/0、优先级=1、描述=`XSG game`
4. 保存。规则会立即生效。

## 腾讯云 CVM / 轻量

1. 控制台 → 云服务器 → 安全组（或 轻量应用服务器 → 防火墙）
2. 「入站规则」→ 「添加规则」
3. 类型=自定义、来源=0.0.0.0/0、协议端口=TCP:3000、备注=`XSG game`
4. 保存。

## 华为云 ECS

1. 控制台 → 弹性云服务器 → 你的实例 → 安全组
2. 「入方向规则」→ 「添加规则」
3. 优先级=1、协议端口=TCP/3000、源地址=0.0.0.0/0、描述=`XSG game`
4. 保存。

## AWS Lightsail / EC2

### Lightsail：
1. 实例 → 网络 → 防火墙 → 「添加规则」
2. 应用程序=自定义、协议=TCP、端口=3000
3. 创建。

### EC2：
1. 实例 → 安全组 → 「编辑入站规则」
2. 类型=自定义 TCP、端口=3000、源=0.0.0.0/0
3. 保存。

## DigitalOcean / Vultr / Linode

实例创建时多半默认放开所有入站。如果配了 `ufw`：
```bash
sudo ufw allow 22/tcp
sudo ufw allow 3000/tcp
sudo ufw enable
sudo ufw status
```

## 一些注意

- 国内云服**只开 3000 不开 80/443 不需要备案**；备案制度只管面向公众的网站。
- 玩家家里有 NAT 没关系，WebSocket 是出方向连接、能上网就能玩。
- 如果想限制只有自己朋友 IP 能连：把 0.0.0.0/0 改成具体 IP 段。但玩家家用 IP 通常会变，可能麻烦。
- 公网 IP 变了一定要：1) 改 RMMZ `XdRs_Online_Net.serverUrl`；2) 重发客户端给朋友。
